Connect Intel — Product & Architecture Overview

Version: June 2026 · Live: https://connectintel.net
Purpose: Share with reviewers. Goal: HubSpot-grade revenue platform (CRM + Marketing + Team ops).

1. What Connect Intel Is

B2B revenue workspace: prospect, qualify, and close leads in one app.

HubWhat users do
CRM / SalesPipeline, lead records, deals, sequences, calendar, WhatsApp
MarketingCampaigns, audiences, templates, automations, forms, analytics
Home / AnalyticsDashboard, team KPIs, activity log, my-day
CollaborationTeam chat (Chithi), notes, tasks
AI ProspectingSearch companies, save to pipeline
WorkspaceTeam invites, permissions, email domain, integrations

North star: HubSpot-style objects, timeline, permissions, reporting — fast SPA, no page reloads.

2. Roles & Data Scoping

RolePipeline access
RepOwn leads only (owner_id = self) — cannot bypass
ManagerTeam by default; “All Departments” → department scope
Org AdminFull org pipeline + team settings, imports, permissions
Marketing rolesManager / Executive / Read-only on marketing hub
Platform AdminConnect Intel operator (customer admin, separate nav)

Scoping enforced in Postgres (pipeline_leads + COUNT RPCs), not in-memory scans.

3. Pages & Navigation (SPA)

Panel-based shell — URL drives panel state, not separate HTML pages.

/home/dashboard              → Home overview
?panel=pipeline&status=new    → CRM pipeline (filtered)
?panel=marketing              → Marketing hub
?panel=team                   → Team & permissions
?lead=<id>                   → Lead record workspace

Layout: Sidebar (nav + scoped lead counts) · Top bar (search, ⌘K) · Panel content · Lead workspace overlay.

Key files: navConfig.js, AppShell.jsx, PipelinePanel.jsx, MarketingPanel.jsx

4. Code Structure

frontend/          React SPA (Vite) → site/
api/index.js       Single Vercel function → all /api/*
lib/server/        Auth, CRM, marketing, store, handlers/
supabase/          Postgres migrations
docs/              Blueprint, release, PRD

Handlers: CORS → requireUser() → business logic → JSON. ~100 API routes.

5. Data & Deployment

Hybrid storage

LayerStorageUse
JSON storestore_collectionsUsers, orgs, lead blobs, marketing metadata
SQL (hot path)Supabase PostgresPipeline queries, RBAC, email queue, activity snapshots

Key tables: pipeline_leads, profiles, teams, departments, marketing_email_queue

Deploy flow

  1. npm run prod:ship (build + checks)
  2. git push origin main
  3. GitHub CI + Vercel → https://connectintel.net
  4. npm run prod:log marks live commit
  5. Supabase migrations run manually in SQL editor

Flags: USE_PIPELINE_LEADS_TABLE, USE_PIPELINE_HIERARCHY_RBAC, USE_MARKETING_SQL_QUEUE
